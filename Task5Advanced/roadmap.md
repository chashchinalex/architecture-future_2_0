# Стратегический Roadmap внедрения Data Mesh

План перехода от монолитного DWH к федеративной архитектуре данных.

## 1. Новые роли и обязанности

Для поддержки Data Mesh необходимо выделить следующие роли:

* **Data Product Owner (в каждом домене):**
    * Отвечает за данные домена (Медицина, Финтех) как за продукт.
    * Определяет SLA, качество данных и доступность для потребителей.
* **Data Engineer (в доменах):**
    * Создает пайплайны (dbt, Kafka Connect) внутри домена.
    * Обеспечивает публикацию данных в Data Lake.
* **Data Platform Engineer (Platform Team):**
    * Отвечает за общую инфраструктуру (Kafka, S3, DataHub).
    * Предоставляет сервисы "как платформу", чтобы домены не настраивали Kafka сами.
* **Data Steward:**
    * Отвечает за Governance, классификацию данных (где ПДн, где открытые данные) и ведение каталога.
